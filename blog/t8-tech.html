Friday: Big O Notation

April 18, 2015

Big O Complexity is what happens as the size of the problem being solved gets larger. Complexity affects performance but not the other way around. Complexity is expressed using Big O Notation.

Big O Notation describes the performance or complexity of an algorithm and uses the worst-case scenario to describe the execution time required by an algorithm. 

O(1) = constant time --> an algorithm that will always excute in the same time or space regardless of the size of the inputs.

Example: A sequence of simple statements

O(N) = linear time --> an algorithm's performance will grow linearly and in direct proportion to the size of the data input set.

Example: For loops and If/Else Statements

O(N^2) = quadratic time --> an algorithm's performance is directly proportional to the square of the size of the input data set.

Example: Nested Loops

O(2^N) = exponential time --> an algorithm's growth will double with each additional element in the input data set.

Example: Recursive computation of Fibonacci Numbers

O(log N) = logarithmic time --> an algorithm's growth peacks at the beginning and slowly flattens out as the size of the data set increases.

Example: Merge or Quick Sort
