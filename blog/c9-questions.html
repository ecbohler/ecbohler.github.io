Wednesday: Asking Good Questions

April 22, 2015

Going down rabbit holes is a part of the learning process especially when learning how to code. However, when we find ourselves face to face with a problem that we can't circumvent, we need to be able to ask for help, but ask for help in a clear and concise way.

There is an art to asking good questions and below are a few key points to help drive this concept home:

  1. Make sure you have a good question title.
  2. Provide some basics i.e. language version, platform version, O.S., etc.
  3. Describe what you've tried and the results. Include results from any research performed as well.
  4. Provide sample code and sample data where possible
  5. Provide screen shots where possible
  6. Proofread for spelling, grammar and formatting.
