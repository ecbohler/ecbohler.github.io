Saturday: Closures - The Bloc, The Proc and The Lambda - A.K.A. Closures

April 4, 2015

Blocks, Procs and Lambdas are closures. Closures are ways of grouping code that you want to run.

Blocks vs. Procs

A Block is a bit of code that can be executed and the syntax uses either do..end or curly braces.

[1,2,3].each { |x| puts x*2}

A Proc is an instance of the Proc class. We can call methods on it and assign it to variables. Procs can also return themselves. Simply stated: a Proc is just a Block that you save to a variable.

p = Proc.new { |x| puts x*2}

[1,2,3].each(&p)

Procs vs. Lambdas

Procs and Lambdas are both Proc objects. However when a function calls a Proc, a return inside that Proc is like a return in the function itself: it takes the control flow out of the function. With a lambda on the other hand, the control flow stays inside the function. Thus a Proc behaves more like a normal block, while a lambda behaves more like a function.
