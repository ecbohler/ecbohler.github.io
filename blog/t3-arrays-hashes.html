Saturday: To Hash or Not To Hash

March 14, 2015

Array = an ordered integer indexed collection of any object

Explanation

  • If you have 1000's of songs
  • Can you remember the index of your favorite song?
  • Arrays aren't good for accessing large amounts of data.

Hash = key-value pairs ndexed via arbitrary keys of any object type

Explanation

  • If you have 1000's of songs
  • You can access the song by artist, genre, etc.
  • Hashes are good for accessing large amounts of data.
